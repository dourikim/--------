Vercel 배포 가이드

로컬 파일 → GitHub → Vercel 순서로 연결하면 매번 파일을 올릴 필요 없이 자동 배포돼요.

최초 설정 (1회만)
1

GitHub 저장소 만들기

github.com/new → 저장소 이름 입력 (예: doouri-website) → Create repository.
VS Code에서 이 프로젝트 폴더를 열고 소스 제어 탭(Ctrl+Shift+G) → "저장소 게시" 클릭.

2

Vercel에 GitHub 연결

vercel.com/new → GitHub으로 로그인 → 저장소 선택 → Deploy.
자동으로 도메인이 생성되고, 설정에서 구입한 도메인을 연결할 수 있어요.

3

도메인 연결

Vercel 프로젝트 → Settings → Domains → 도메인 입력 → DNS 설정 안내에 따라 도메인 레지스트라에서 설정.
보통 24시간 이내 연결 완료.

평소 배포 방법 (업데이트할 때마다)
1

파일 수정 후 저장

로컬에서 HTML/CSS/JS 파일 수정하고 저장.

2

VS Code에서 커밋 & 푸시

소스 제어 탭(Ctrl+Shift+G) → 변경 파일 확인 → 메시지 입력 (예: "수업 페이지 업데이트") → ✓ 커밋 → 동기화(↑↓).
또는 터미널에서: git add . && git commit -m "업데이트" && git push

3

Vercel이 자동 배포

push하면 Vercel이 자동으로 감지해서 약 30초~1분 내 라이브 반영.
vercel.com/dashboard에서 배포 상태 확인 가능.

★

Draft 페이지 미리보기 방법

VS Code에서 새 브랜치 만들기 (git checkout -b draft/인강페이지) → 수정 → push.
Vercel이 자동으로 미리보기 URL 생성 (예: draft-인강페이지--doouri.vercel.app).
확인 후 main 브랜치에 병합하면 라이브 반영. 브랜치 = Draft main = Live
