03여백 시스템 (8px 그리드)

Spacing Scale — CSS Variables

--sp-4
4px 아이콘-텍스트 간격, 마이크로 간격
--sp-8
8px 카드 내부 항목 간격, 칩 간격
--sp-16
16px 카드 패딩, 섹션 내부 여백
--sp-24
24px 좌우 페이지 패딩 (모바일)
--sp-32
32px 섹션 패딩 상하, .section-block
--sp-64
64px 히어로 상단 패딩, 큰 섹션 여백
--sp-128
128px 페이지 하단 여백, 대형 분리
8px 배수 규칙: 모든 여백과 크기 값은 4, 8, 16, 24, 32, 64, 128의 배수를 사용합니다. 이 외의 값(예: 15px, 22px)은 사용하지 않습니다. 단, border-radius(10px), line-height(비율값)는 예외.

05적용 원칙

제목 = 고운바탕, 본문 = 프리텐다드

두 서체를 같은 역할에 혼용하지 않습니다. 서체 자체가 위계를 만듭니다.

Brick Red는 한 화면에 하나의 핵심 포인트

버튼, 강조 텍스트, 아이콘 등에 동시에 남발하지 않습니다. 많을수록 강조 효과가 희석됩니다.

여백은 8px 배수만 사용

새로운 padding/margin을 추가할 때 반드시 CSS 변수(--sp-*)를 사용하거나 8의 배수 값을 씁니다.

모바일 우선 (Mobile First)

기본 스타일은 모바일 기준. 데스크톱 레이아웃은 @media (min-width: 769px)에서 추가합니다.

색상 변경은 :root 변수만 수정

하드코딩된 hex 값을 직접 쓰지 않습니다. 변경이 필요하면 style.css 상단의 :root 블록에서만 수정합니다.
